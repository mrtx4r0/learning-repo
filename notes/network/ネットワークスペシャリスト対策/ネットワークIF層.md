# データリンク
* PC-ルータ、ルータ-サーバなどの機器間の通信単位
* フレームという単位でのデータのやりとり
*  MACアドレスで相手を識別
*  代表的な有線LAN規格はイーサネット、FDDI、トークンリング
*  OSI基本参照モデルでいうデータリンク層（第2層）は、さらに論理リンク制御/媒体アクセス制御という2つ副層に分けられる（＝設計上・標準化上の抽象概念）<br>
   `論理リンク制御(LLC)`：イーサネット、FDDIなどネットワーク媒体に依存しないデータリンク層共通のヘッダ制御(=やルール)。<br>
   `媒体アクセス制御(MAC)`：イーサネット、FDDIなどイーサネットごとに決まっているヘッダ制御。具体的な制御の方式として、CSMA/CD、トークンパッシング等がある<br>
*  媒体共有型/媒体非共有型
*  半二重通信/全二重通信

## イーサネット
* 現在最も使用されている**LANの規格**。物理層、データリンク層を規定
* IEEE802: IEEE学会のうち、LAN規格を定めている委員会
* IEEE802.3: イーサネットLANに関する規格（またはワーキンググループ）
* 現在普及している「イーサネットLAN」はDEC、Intel、Xeroxが作ったEthernetⅡ 規格を元に、1983年にIEEE802.3 CSMA/CDとして策定された仕様
* 伝送メディア(有線)
  * 同軸ケーブル、UTP、STP、光ファイバーケーブル
    * 同軸ケーブル: 10base2、10base5で使用された。現在のLANでは使用されてない。
    * UTP: 現在のLANの主流。uはunshielded。安価だがノイズに弱い
    * STP: sはshielded。ノイズに強く工場などで使用
    * 光ファイバー: コンピュータの電気信号をレーザーを使用して光信号に変換、生成されたレーザー光を光ファイバーケーブルに通してデータの送受信を行う
* イーサネットヘッダ(現在主流なEthernetⅡのフレームフォーマット※)
  * 宛先MACアドレス: 48bit
  * 送信元MACアドレス: 48bit
  * タイプ(EtherType): 16bit。上位層(ネットワーク層)のプロトコルが何であるかを示すフィールド。
    * 0800: Ipv4
    * 0806: ARP
    * 86DD: Ipv6
  * FCS: 16bit<br>
※IEEE802.3として標準化されたフレームフォーマットは、EtherTypeではなく、Length(データ長を示す)/LLC
/SNAP(用途はEtherTypeと同じ)のフィールドを持つが、実装として主流なのはEthernetⅡの方。IEEE802.3はどちらのフレームフォーマットも許容している)

## 無線LAN
* IEEE 802.11: 無線LAN規格。CSMA/CA方式を採用。
* アドホックモード: IEEE 802.11の動作モードのひとつ。端末同士が直接通信
* インフラストラクチャーモード: IEEE 802.11の動作モードのひとつ。アクセスポイントを経由して通信
* RTS/CTS方式: 無線LANの「隠れ端末問題」を解決するためのフレーム制御の方式。
* 近距離無線通信: IEEE 802.15で規格をまとめている。代表的規格はbluetooth、BLE、ZigBee、Wi-Sun 

## PPP
* コネクション型通信。データリンク層だけのプロトコル。
* 専用線、PSTN、ISDNなどのポイントツーポイント接続で最もよく使用される（使用された。今はISDNなどは使われない）
* `HDLC`をベースに開発された
* プロトコルフィールド（ProtocolType）があるためIP,IPXなど複数プロトコルを使用した環境をサポート
* PPPはオプションとして、**認証**、**圧縮**、**マルチリンク**、**エラー制御***をサポート
* `LCP`(Link Control Protocol) と `NCP`（Network Control Protocol）の2つのプロトコルから構成される
* **LCP、NCPはPPPフレームのペイロード(データ)部分に入る(=カプセル化される)**
* PPPの接続は、以下の順で行われる
  * ① `LCP`におけるリンク確立
  * ② 認証（認証がある場合のみ）
  * ③ `NCP`における接続の確立
* `LCP`: PPPのうち上位プロトコルに依存しないプロトコル。リンクの確立・維持、ユーザ認証を行う
* `NCP`: PPPのうち上位プロトコルに依存するプロトコル<br>
* `PAP`、`CHAP`: LCPにてリンク確立後に行う認証のプロトコル。LCPがオプションとしてサポートするプロトコル
* PPPフレームフォーマットの主なフィールド
  * フラグシーケンス: PPPフレームの前後に付与される`01111110`という1が6つ連続する8ビットパターン。データの開始終了を表す(同期をとるため)
  * アドレス: 常にブロードキャストであるため`11111111`
  * タイプ(Protocol): 上位層のプロトコルを示す値が入る
    * 0xC021: LCP
    * 0x8021: IPCP(IP用NCP)
    * 0xC023: PAP
    * 0xC223: CHAP
    * 0x0021: IPv4
    * 0x0057: IPv6
* **`PPPoE`**: イーサネットにPPPの認証・コネクション管理・課金管理機能を導入したプロトコル。PPPフレームがイーサネットフレームにカプセル化される<br>

## その他データリンク
### FDDI
* **トークンパッシング方式(アペンドトークン方式)**
### ATM
* Asynchronous Transfer Mode
* 53オクテットの固定長(セルという単位)でデータを伝送する。固定長のため高速
### IEEE1394
* 高速シリアルバス
* ほぼUSBにとって代わられた
### HDMI
* バージョン1.4でイーサネットフレームをデする規格が追加されたことにより、HDMIケーブルでTCP IP通信が可能となった。ただしネットワーク用途での使用は少ない。
