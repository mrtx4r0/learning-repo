# LAN間接続
LAN同士を接続する主な装置・技術
## スイッチングハブ
* MACアドレス学習機能とフィルタリング(必要なポートにのみフレームを転送する)機能<br>
  ※ スイッチングハブ登場前のリピータハブは、全ポートにフレームを流すだけ
* **ポートミラーリング**
  * 前提<br>
  フィルタリング機能により、他ポート宛てのフレームは見えず、ネットワーク管理上、自ポート宛て以外のフレームも監視できる必要がある(逆にフィルタリング機能がない時代は、自分宛以外のフレームもtcpdumpで見れた)
  * どんな機能か？<br>
    適当な空きポートを**ミラーポート**、コピーしたい監視対象のポート(src)として設定することで、ミラーポートに接続した監視用PCにコピーしたフレームを流すことができる
* **オートネゴシエーション**<br>
※前提<br>
接続するスイッチ同士、スイッチとPC間では、お互いがどんな通信速度/通信方式(半二重通信/全二重通信)に対応しているのか認識を合わせておく必要がある<br>
  * オートネゴシエーションは、どんな通信速度/通信方式で通信するか？を機器同士で通信して自動で決める機能。物理層よりの仕組みで、特殊な信号で通信する
  * 固定モード: 通信速度/通信方式を固定で決めるモード。
  * 固定モード/オートネゴシエーションを切り替えられる製品が主流。オートネゴシエーションでの使用が多い
  * 機器同士でオートネゴシエーション/固定モードのどちらかに合わせないと不具合が発生することがある
  * linuxでは、以下コマンドでNICの設定(オートネゴシエーションかどうかを含む情報)くが見れる
  ```bash
  $ ethtool eth0(デバイス名)
  ```

## VLAN（Virtual LAN）
※前提<br>
もともとスイッチは、物理的に同じスイッチに接続している機器は同じLANセグメントに属することになる<br>
* スイッチでの物理的な接続形態とは独立して、**仮想的なLANセグメントを作る**技術
* VLANを使用することで、L2スイッチでルータやL3スイッチのようにブロードキャストドメインを分割することが可能
* スイッチのポートごとに**VLAN ID**を設定することで、同じIDごとにLANセグメント化することが可能
    (同じ192.0.2.0/24のネットワーク内でも、VLAN IDを分けて分割できる）<br>
* 業務用途では、VLAN対応のスイッチングハブの使用が主流（部署間などでネットワークを分けたいため）
* VLANの設定は、手動（コマンド）でやるスタティックVLAN、動的にやるダイナミックVLANがある
