# IP
* IPヘッダ (下位ビットから)<br>
  * フラグ(flags) : &nbsp;3bit。パケット分割の制御情報。パケットの分割を許可するか/分割された最後or途中のフラグメントか
  * 生存期間(TTL) : &nbsp;8bit。パケットの生存期間を示す。1台のルータを通過するごとにTTL値は1ずつデクリメントされ、0になるとパケットが破棄される
  * プロトコル(protocol) : &nbsp;8bit。上位層(トランスポート層)のプロトコルが何であるかを示す。ICMP, OSPFなど同じ階層のプロトコルを示す場合もある。
  * 送信元IPアドレス(source address) : &nbsp;32bit
  * 宛先IPアドレス(destination address) : &nbsp;32bit

# ARP
* ARPパケット: &nbsp;
  * IPパケットではない(ARPパケットのデータとしての送信元/宛先IPアドレスのフィールドがある)。
  * イーサネットフレームにカプセル化される(=L2ヘッダが付加される)
* **ARPキャッシュ、ARPテーブル**: &nbsp;ARP通信でやりとりされた情報は、一定時間ARPテーブルにARPキャッシュとして保持される
* **GARP**: &nbsp;ARPパケットの一種。
  * ①自分自身に設定するIPアドレスが重複していないか否かの検出
  * ②同一セグメントのネットワーク機器上の**ARPキャッシュを更新させる**という2つの役割。<br>
    参考→ https://www.infraexpert.com/study/tcpip3.5.html

# ルーティング
## ルータ
* ルーティングテーブルを管理し、宛先IPアドレスを基に適切なネットワークにパケットを転送する
* NAT/NAPT機能、DHCP機能、アクセスポイント機能などもある
* AS:ルーティングポリシを決めてそれを適用して運用する範囲
* IGP:AS内で使うルーティングプロトコルの総称
* EGP:AS間で使うルーティングプロトコルの総称

## ルーティングプロトコル
* 前提となる概念
  * **AS**（Autonomous System）
    * 1つのISP、企業、研究機関など共通ポリシーにより運用されるネットワークの集合体
    * ASごとに一意のIDが割り当てられており、IANAが管理・日本ではJPNICが割り当てを行う
  * コンバージェンス（収束）
    * ネットワーク上の全てのルータが、ルーティングテーブルに本来あるべき必要な宛先ルートを安定的に保持している状態
* ルーティングの方法
  * 直接接続
  * スタティックルーティング
  * ダイナミックルーティング
* ダイナミックルーティング
  * IGPとEGP
    * BGP＝AS間のプロトコルの総称、IGP＝AS内のプロトコルの総称
    * IGP（Interior Gateway Protocol）
     * RIP、EIGRP、OSPF、IS-IS
    * EGP（Exterior Gateway Protocol）
      * BGP
* IGPのアルゴリズムによる分類
  * 参考：https://www.infraexpert.com/study/routing6.html
  * ディスタンスベクター/リンクステート/ハイブリッド
    * ディスタンスベクター
      * 定期的にルータ間でルーティングテーブル全体を交換し合い、ホップ数により最適ルートを決定する
      * RIPv1、RIPv2が該当
    * リンクステート
      * OSPF、IS-ISが該当
    * ハイブリッド
      * EIGRPが該当
* IGPのクラスフル/クラスレスによる分類
