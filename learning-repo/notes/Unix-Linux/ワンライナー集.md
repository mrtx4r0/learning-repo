# 条件分岐
```bash
# $ STR="TRUE"
$ if [ "$STR" = "TRUE" ]; then echo "STR="$STR""; else echo "FALSE"; fi
# 実行結果
# STR=TRUE
```

# テキストファイルに書かれている名前の空ファイルを作成する
以下のようなテキストファイルを作成
```bash
# test.txtの内容
A_0002C0000_20251026_3W200125W.MOV
```
コマンド
```bash
$ cat test.txt | xargs touch
# $ ls ~/work # ファイルが作成されていることを確認する
# A_0002C0000_20251026_3W200125W.MOV
```

# 任意のディレクトリ配下の特定の拡張子のファイルを削除する
注）サブディレクトリ内の該当ファイルも削除する

パターン1：findを使う
```bash
$ find ~/work/_old -type f -name "*.txt" -exec rm {} \;
```


パターン2：rmを使う
```bash
$ shopt -s globstar # グロブスター「**」を有効にする
$ rm ~/work/_old/**/*.txt
```
グロブスター「**」は、0個以上のディレクトリに再帰的にマッチする

# シンボリックリンクのリンク先を変える
現在/Users/username/dotfiles/.zshrcを指しているシンボリックリンク/Users/username/.zshrcのリンク先(実体ファイルの場所)を変える
```zsh
ln -nfs /Users/username/github/dotfiles/.zshrc
/Users/username/.zshrc
```
ln -fs <新しいリンク先> <シンボリックリンク>

#### -f(--force)
<シンボリックリンク>が既に存在している場合、それを削除して新規にリンクを作る

#### <u>-n(--no-difference)</u>
<シンボリックリンク>が、ディレクトリでも、ファイル名として扱う
例えば、
```zsh
ln -s foo.txt bar/dir #dirはディレクトリ
```
の場合、シンボリックリンクbar/dir/foo.txtが作られるが、
```zsh
ln -s foo.txt bar/dir
```
の場合、bar/dirというシンボリックリンクが作られる\(**つまり指定した名前通りにリンクを作ることができる**\)

# psの出力結果をgrepする
USER(auxの出力1行目)またはsystemdを含む行を抽出
```bash
$ ps aux | grep -E '(^USER|systemd)'
```

# tar.bz2を展開する
```bash
$ tar -xvjf busybox-1.37.0.tar.bz2
```
`-x`: 展開する(extract)
`-v`: 展開ファイルを表示(verbose)
`-j`: bzip2形式
`-f`: ファイル指定
